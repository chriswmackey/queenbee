Examples¶

Minimal¶

The minimal configuration for an operator can be found below. The keys indicated here are the ones you absolutely have to fill in for this operator to be validated by Queenbee.

metadata:
  name: minimum-viable-operator
  tag: 0.0.1
config:
  docker:
    image: some/docker-image:latest
    workdir: /some/absolute/directory
functions: []

Fully Configured¶

The operator below shows example values for every possible key in the Operator object.

metadata:
  name: full-operator
  tag: 0.0.1
  app_version: 3.4.3
  description: A test operator with every possible bit of metadata included
  keywords:
  - test
  - full
  maintainers:
  - name: ladybugbot
    email: ladybugbot@ladybug.tools
  home: https://ladybug.tools/full-operator
  sources:
  - https://github.com/ladybug-tools/full-operator
  - https://gihtub.com/ladybug-tools/queenbee
  icon: https://www.ladybug.tools/assets/img/logo.png
  deprecated: false
config:
  docker:
    image: some/docker-image:latest
    registry: https://docker.pkg.github.com # Github Registry
    workdir: /some/absolute/directory
functions:
- name: minimum-function
  command: echo woop

Energy Plus¶

This operator is the one created when following the Operator Creation Guide.

metadata:
  name: energy-plus
  tag: 0.1.0
  app_version: 9.0.1   # The version of EnergyPlus the operator uses
  description: An operator to run EnergyPlus functions
  keywords:
  - energyplus
  home: https://energyplus.net
  sources:
  - https://github.com/nrel/energyplus
  icon: https://energyplus.net/sites/default/files/eplus_logo.png

config:
  docker:
    image: nrel/energyplus:9.0.1
    workdir: /var/simdata

functions:
- name: design-day-simulation
  description: Run an EnergyPlus Design Day simulation using a weather file and an idf
  inputs:
    artifacts:
    - name: idf
      path: input.idf
  command: energyplus -D -x -d simulation-outputs input.idf
  outputs:
    artifacts:
    - name: outputs
      path: simulation-outputs
- name: run-simulation
  description: Run an EnergyPlus simulation using a weather file and an idf
  inputs:
    artifacts:
    - name: weather
      path: weather.epw
    - name: idf
      path: input.idf
  command: energyplus -w weather.epw -x -d simulation-outputs input.idf
  outputs:
    artifacts:
    - name: outputs
      path: simulation-outputs

Honeybee Radiance¶

This is an example operator called honeybee-radiance. This operator uses the honeybee-radiance CLI in a Docker container which has radiance installed on it. Each function is templated with parameter inputs and explicitely indicates the artifacts (files) it expects to find at a certain path.

Note

The app_version matches the docker container release tag.

metadata:
  name: honeybee-radiance
  description: An operator to run all things honeybee and radiance!
  icon: https://www.ladybug.tools/assets/img/logo.png
  home: https://ladybug.tools/honeybee-radiance/docs
  sources:
  - https://github.com/ladybug-tools/honeybee-radiance

  tag: 1.2.3
  app_version: 2.3.2

  keywords:
  - honeybee
  - radiance
  - ladybug-tools

  maintainers:
  - name: mostapha
    email: mostapha@ladybug.tools

config:
  docker:
    image: ladybugtools/honeybee-radiance:2.3.2
    workdir: /opt/run/

functions:
- name: create-octree
  description: Generate an octree
  inputs:
    artifacts:
    - name: model
      path: model.json
    - name: sky
      path: sky.sky
  command: honeybee radiance oconv -s sky.sky -m model.json --output scene.oct
  outputs:
    artifacts:
    - name: scene-file
      path: scene.oct

- name: 10000-lux-sky
  description: Generates a 10000 lux sky
  command: honeybee radiance sky illuminance 100000
  outputs:
    artifacts:
    - name: sky
      description: The resulting sky object
      path: sky.sky

- name: post-process
  description: Post process your results and $$$
  inputs:
    artifacts:
    - name: raw-results-folder
      path: raw
  command: honeybee radiance grid merge raw --folder output
  outputs:
    artifacts:
    - name: post-process-folder
      path: output
    parameters:
    - name: daylight-factor-max
      path: output/max
    - name: daylight-factor-average
      path: output/avg

- name: ray-tracing
  description: Run ray tracing using some input data!
  inputs:
    parameters:
    - name: radiance-parameters
      description: a string of radiance parameters
      default: -b 5
    artifacts:
    - name: grid
      path: grid.pts
    - name: scene-file
      path: scene.oct
  command: rtrace -I -h {{inputs.parameters.radiance-parameters}} scene.oct < grid.pts > grid.res
  outputs:
    artifacts:
    - name: result-file
      path: grid.res
      
- name: split-grid
  inputs:
    parameters:
    - name: grid-count
      required: true
    artifacts:
    - name: grid
      description: The input grid to split
      path: grid.pts
  command: honeybee radiance grid split grid.pts --folder output --log-file output/grids.txt
  outputs:
    parameters:
    - name: grid-list
      path: output/grids.txt
    artifacts:
    - name: output-grids-folder
      path: output
